Microsoft 安全公告 MS10-058 - 重要
==================================

TCP/IP 中的漏洞可能允许特权提升 (978886)
----------------------------------------

发布时间: 2010年8月10日 | 更新时间: 2012年3月13日

**版本:** 2.0

### 一般信息

#### 摘要

此安全更新可解决 Microsoft Windows 中两个秘密报告的漏洞。由于处理特定输入缓冲区时发生错误，较严重的漏洞可能允许特权提升。能够登录目标系统的攻击者可以利用此漏洞，并使用系统级别的特权运行任意代码。攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。

对于 Windows Vista、Windows Server 2008、Windows 7 和 Windows Server 2008 R2 的所有受支持版本，此安全更新的等级为“重要”。有关详细信息，请参阅本节的“**受影响和不受影响的软件**”小节。

通过纠正 TCP/IP 堆栈处理错误的 IPv6 数据包以及通过用户模式复制的数据的方式，安全更新解决了漏洞问题。有关这些漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的“常见问题 (FAQ)”小节。

**建议。** 大多数客户均启用了“自动更新”，他们不必采取任何操作，因为此安全更新将自动下载并安装。尚未启用“自动更新”的客户必须检查更新，并手动安装此更新。有关自动更新中特定配置选项的信息，请参阅 [Microsoft 知识库文章 294871](http://support.microsoft.com/kb/294871)。

对于管理员、企业安装或者想要手动安装此安全更新的最终用户，Microsoft 建议客户使用更新管理软件尽早应用此更新或者利用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服务检查更新。

另请参阅本公告后面部分中的“**检测和部署工具及指导**”一节。

**已知问题。** 无

IPv6 内存损坏漏洞 - CVE-2010-1892  
---------------------------------
  
   
由于处理格式错误的扩展头的特制 IPv6 数据包中有错误，Microsoft Windows 中的 TCP/IP 处理中存在一个拒绝服务漏洞。攻击者可能利用漏洞向目标系统发送为数不多的特制数据包，从而导致受影响的系统停止响应。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2010-1892](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-1892)。
  
#### IPv6 内存损坏漏洞 (CVE-2010-1892) 的缓解因素
  
Microsoft 并未发现此漏洞的任何缓解因素。
  
#### IPv6 内存损坏漏洞 (CVE-2010-1892) 的变通办法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。Microsoft 已测试了以下变通方法，并在讨论中指明了变通方法是否会降低功能性：
  
-   **禁用 Internet 协议版本 6 (IPv6) 组件**
  
    **注意**请参阅[Microsoft 知识库文章 929852](http://support.microsoft.com/kb/929852)以使用自动**Microsoft Fix it**解决方案 50444 禁用 IPv6 组件（环回接口除外）。此修复解决方案将要求完成时重新启动才能生效。此修复解决方案部署变通办法，因此具有相同的用户影响。我们建议管理员在部署此修复解决方案之前仔细阅读知识库文章。
  
    **注意** 如果注册表编辑器使用不当，可能会导致严重的问题，可能必须重新安装操作系统。Microsoft 不保证能够解决因注册表编辑器使用不当而导致的问题。使用注册表编辑器的风险由您自己承担。有关如何编辑注册表的信息，请查看注册表编辑器 (Regedit.exe) 中的“更改项和值”帮助主题，或查看 Regedt32.exe 中的“在注册表中添加和删除信息”和“编辑注册表数据”帮助主题。
  
    1.  单击“**开始**”，在“**开始搜索**”框中键入 **regedit**，然后在“**程序**”列表中单击 **regedit.exe**。  
    2.  在“**用户帐户控制**”对话框中，单击“**继续**”。  
    3.  在注册表编辑器中，找到并随后单击下列注册表子项：  
        ```
        HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip6\Parameters\
        ```  
    4.  双击 **DisabledComponents** 修改 DisabledComponents 项。  
        **注意** 如果 DisabledComponents 项不可用，您必须创建它。为此，可执行以下步骤：  
        a. 在“**编辑**”菜单上，指向“**新建**”，然后单击“**DWORD（(32 位）值**”。  
        b. 键入 **DisabledComponents**，然后按 **ENTER**。  
        c.　双击 **DisabledComponents。**  
    5.  输入 **0xffffffff** 以禁用所有 IPv6 组件，IPv6 环回接口除外。此值也将 Windows 配置为在前缀策略中使用 Internet 协议版本 4 (IPv4) 而不是 IPv6。  
    6.  单击“**确定**”。您必须重新启动计算机，这些更改才会生效。
  
    **变通办法的影响。**此更改生效之后，路由和远程访问服务将失败。
  
    **如何撤消变通方法。**
  
    **注意**请参阅[Microsoft 知识库文章 929852](http://support.microsoft.com/kb/929852)以使用自动**Microsoft Fix it**解决方案 50440 启用 IPv6。该 Fix it 解决方案要求在完成时重新启动才会生效。
  
    1.  单击“**开始**”，在“**开始搜索**”框中键入 **regedit**，然后在“**程序**”列表中单击 **regedit.exe**。  
    2.  在“**用户帐户控制**”对话框中，单击“**继续**”。  
    3.  在注册表编辑器中，找到并随后单击下列注册表子项：  
        ```
        HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip6\Parameters\
        ```  
    4.  删除 **DisabledComponents** 项。  
    5.  您必须重新启动计算机，这些更改才会生效。
  
#### IPv6 内存损坏漏洞 (CVE-2010-1892) 的常见问题解答
  
**此漏洞的影响范围有多大？**  
这是一个拒绝服务漏洞。成功利用此漏洞的攻击者可能会导致受影响系统停止响应。请注意，虽然攻击者无法利用拒绝服务漏洞来执行代码或提升他们的用户权限，但此漏洞可能导致受影响的系统停止接受请求。
  
**造成此漏洞的原因是什么？**  
漏洞是由于 Windows 网络堆栈未正确处理特定的错误数据包造成的。
  
**什么是 TCP/IP？**  
TCP/IP 是一套在 Internet 上广泛应用的网络协议。TCP/IP 在相互连接的计算机网络间提供通信，这些计算机有着不同的硬件结构并运行了不同的操作系统。TCP/IP 包含了计算机通信的标准及网络连接和路由通信的惯例。有关 TCP/IP 的详细信息，请参阅 TechNet 文章[网络和 TCP/IP 概述](http://technet.microsoft.com/en-us/library/cc739443(ws.10).aspx)。
  
**什么是 IPv6？**  
Internet 协议版本 6 (IPv6) 是一套新的 Internet 网络层标准协议，内置于 Microsoft Windows XP 和更高版本中。IPv6 旨在解决 IP 当前版本 IPv4 的许多问题，例如地址耗尽、安全性、自动配置和扩展性。要了解有关 IPv6 的更多信息，请阅读以下 TechNet 站点的 [IPv6 常见问题](http://technet.microsoft.com/en-us/network/bb530961.aspx)。
  
**攻击者可能利用此漏洞执行什么操作？**  
成功利用此漏洞的攻击者可能导致受影响的系统无响应。
  
**攻击者如何利用此漏洞？**  
通过发送少量特制的错误 IPv6 数据包到目标系统，攻击者可以利用此漏洞。
  
**受此漏洞威胁最大的系统有哪些？**  
所有启用 IPv6 功能的操作系统均受此漏洞威胁。默认情况下，Windows Vista 和 Windows Server 2008 中启用了所有受支持的 IPv6 功能。
  
**此更新有什么作用？**  
通过纠正 Windows TCP/IP 堆栈处理格式错误的 IPv6 数据包的方式，更新解决了此漏洞问题。

Windows 网络中的整数溢出漏洞 – CVE-2010-1893  
--------------------------------------------
  
   
由于处理特定输入缓冲区时发生错误，Microsoft Windows 中的 TCP/IP 处理中存在一个特权提升漏洞。成功利用此漏洞的攻击者可以使用系统级特权运行任意代码。攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2010-1893](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-1893)。
  
#### Windows 网络中整数溢出漏洞的缓解因素 – CVE-2010-1893
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   攻击者必须拥有有效的登录凭据并能本地登录才能利用此漏洞。匿名用户无法利用此漏洞，也无法以远程方式利用此漏洞。
  
#### Windows 网络中整数溢出漏洞的变通办法 – CVE-2010-1893
  
Microsoft 并未发现此漏洞的任何变通办法。
  
#### Windows 网络中整数溢出漏洞的常见问题解答 – CVE-2010-1893
  
**此漏洞的影响范围有多大？**  
这是一个特权提升漏洞。成功利用此漏洞的攻击者可执行任意代码，并可完全控制受影响的系统。攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**造成此漏洞的原因是什么？**  
漏洞是由于 Windows TCP/IP 堆栈未正确处理通过用户模式复制的数据，引发整数溢出导致的。
  
**什么是 TCP/IP？**  
TCP/IP 是一套在 Internet 上广泛应用的网络协议。TCP/IP 在相互连接的计算机网络间提供通信，这些计算机有着不同的硬件结构并运行了不同的操作系统。TCP/IP 包含了计算机通信的标准及网络连接和路由通信的惯例。有关 TCP/IP 的详细信息，请参阅 TechNet 文章[网络和 TCP/IP 概述](http://technet.microsoft.com/en-us/library/cc739443(ws.10).aspx)。
  
**什么是 IPv6？**  
Internet 协议版本 6 (IPv6) 是一套新的 Internet 网络层标准协议，内置于 Microsoft Windows XP 和更高版本中。IPv6 旨在解决 IP 当前版本 IPv4 的许多问题，例如地址耗尽、安全性、自动配置和扩展性。要了解有关 IPv6 的更多信息，请阅读以下 TechNet 站点的 [IPv6 常见问题](http://technet.microsoft.com/en-us/network/bb530961.aspx)。

**攻击者如何利用此漏洞？**  
要利用此漏洞，攻击者必须先登录到系统。然后，攻击者可以运行一个为利用此漏洞而特制的应用程序，从而完全控制受影响的系统。这只是一个本地漏洞，无法从远程进行攻击。
  
**受此漏洞威胁最大的系统有哪些？**  
终端服务器等系统尤其会受此漏洞的影响，用户可在其中运行不受信任的代码。
  
**此更新有什么作用？**  
通过纠正 Windows TCP/IP 堆栈处理通过用户模式复制的数据的方式，更新解决了此漏洞问题。
