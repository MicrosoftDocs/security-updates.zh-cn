Microsoft 安全公告 MS07-064 - 严重
==================================

DirectX 中的漏洞可能允许远程执行代码 (941568)
---------------------------------------------

发布时间: 2007年12月11日

**版本:** 1.0

### 一般信息

#### 摘要

此严重安全更新可解决 Microsoft DirectX 中两个秘密报告的漏洞。 如果用户在 DirectX 中打开用于流式媒体的特制文件，这些漏洞可能允许执行代码。 如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。

这是 Microsoft Windows 2000、Windows XP、Windows Server 2003 和 Windows Vista 的所有受支持版本的严重安全更新。 有关详细信息，请参阅本节中“**受影响和不受影响的软件**”小节。

有关这些漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的“常见问题 (FAQ)”小节。

**建议。**  Microsoft 建议用户立即应用此更新。

**已知问题。**  [Microsoft 知识库文章 941568](http://support.microsoft.com/kb/941568) 介绍了客户在安装此安全更新时可能遇到的当前已知问题。 本文还介绍了这些问题的建议解决办法。

#### 受影响和不受影响的软件

已对下列软件进行测试，以确定受到影响的版本。 其他版本的支持生命周期已结束或者不受影响。 要确定软件版本的技术支持生命周期，请访问 [Microsoft 技术支持生命周期](http://go.microsoft.com/fwlink/?linkid=21742)。

Microsoft DirectX 在 SAMI 文件分析中的代码执行漏洞 - CVE-2007-3901  
------------------------------------------------------------------
  
  
DirectX 处理受支持格式文件的方式中存在一个远程执行代码漏洞。 如果用户打开特制文件，此漏洞可能允许执行代码。 如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2007-3901](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3901)。
  
#### Microsoft DirectX 在 SAMI 文件分析中的代码执行漏洞 (CVE-2007-3901) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   此漏洞无法在用户查看或预览电子邮件时通过电子邮件自动加以利用。 用户必须打开电子邮件附件，以电子邮件为载体的攻击才会得逞。  
-   此漏洞无法通过 Windows 2000 Service Pack 4 上的 Windows Media Player 6.4 的基于 Web 的攻击情形加以利用。
  
#### Microsoft DirectX 在 SAMI 文件分析中的代码执行漏洞 (CVE-2007-3901) 的变通办法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。 Microsoft 已测试了以下变通方法，并在讨论中指明了变通方法是否会降低功能性：
  
-   **修改 quartz.dll 的访问控制列表**
  
    在 Windows XP（所有版本）上，通过命令提示符运行以下命令：
  
    `Echo y| Cacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /E /P everyone:N`
  
    在 Windows Vista（所有版本）上，通过提升的命令提示符运行以下命令：
  
    `Takeown.exe /f %WINDIR%\SYSTEM32\QUARTZ.DLLIcacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /save %TEMP%\QUARTZ_ACL.TXTIcacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /deny everyone:(F)`
  
**变通办法的影响：**  WAV 和 AVI 文件将无法在 Windows Vista 上的启用 DirectX 的应用程序中播放。 所有文件将无法在 Windows XP 上的启用 DirectX 的应用程序中播放 **。**
  
**如何撤消变通方法：** 在 Windows XP（所有版本）上，通过命令提示符运行以下命令：
  
   `Cacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /E /R everyone`
  
在 Windows Vista（所有版本）上，通过提升的命令提示符运行以下命令：
  
`Icacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /grant everyone:(F)Icacls.exe %WINDIR%\SYSTEM32 /restore %TEMP%\QUARTZ_ACL.TXT`
  
-   **注销 quartz.dll**
  
    `Regsvr32.exe –u %WINDIR%\SYSTEM32\QUARTZ.DLL`
  
**变通办法的影响：**  WAV 和 AVI 文件将无法在 Windows Vista 上的启用 DirectX 的应用程序中播放。 所有文件将无法在 Windows XP 上的启用 DirectX 的应用程序中播放。
  
**如何撤消变通方法：** 通过提升的命令提示符运行以下命令：
  
`Regsvr32.exe %WINDIR%\SYSTEM32\QUARTZ.DLL`
  
#### Microsoft DirectX 在 SAMI 文件分析中的代码执行漏洞 (CVE-2007-3901) 的常见问题
  
**此漏洞的影响范围有多大？**    
这是一个远程执行代码漏洞。 成功利用此漏洞的攻击者可以完全远程控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**造成漏洞的原因是什么？**    
Microsoft DirectShow 是集成的 DirectX 技术，不执行对 Synchronized Accessible Media Interchange (SAMI) 文件类型参数的充分分析。
  
**什么是 DirectShow？**    
Microsoft DirectShow 用于 Microsoft Windows 操作系统上的流式媒体。 DirectShow 用于高质量地捕获和播放多媒体流。 它自动检测和使用视频和音频加速硬件（如果有），同样也支持没有加速硬件的系统。 DirectShow 还与其他 DirectX 技术集成。 可以使用 DirectShow 创建的应用程序的一些示例包括 DVD 播放器、视频编辑应用程序、AVI 到 ASF 转换器、MP3 播放器和数字视频捕获应用程序。
  
**什么是 DirectX？**    
Microsoft DirectX 是 Windows 操作系统的一个功能。 它用于 Microsoft Windows 操作系统上的流式媒体，在玩游戏或观看视频时支持图形和声音。
  
**攻击者可能利用此漏洞执行什么操作？**    
如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**攻击者如何利用此漏洞？**    
利用此漏洞需要用户打开特制格式文件。 但是，由于此漏洞在 Microsoft Windows 的流式组件中，攻击可以从特制网站或提供 Web 内容的任何应用程序发起。
  
**受此漏洞威胁最大的系统有哪些？**

Microsoft DirectX 在 WAV 和 AVI 文件分析中的代码执行漏洞 - CVE-2007-3895  
------------------------------------------------------------------------
  
  
DirectX 处理 WAV 和 AVI 格式文件的方式中存在一个远程执行代码漏洞。 如果用户访问特制网站或打开包含特制内容的电子邮件，此漏洞可能允许执行代码。 如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2007-3895](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3895)。
  
#### Microsoft DirectX 在 WAV 和 AVI 文件分析中的代码执行漏洞 (CVE-2007-3895) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   在基于 Web 的攻击情形中，攻击者可能拥有一个网站，并在上面放置用来利用此漏洞的网页。 另外，接受或宿主用户提供的内容或广告的网站以及受到破坏的网站可能包含可能利用此漏洞的特制内容。 但是在所有情况下，攻击者无法强制用户访问这些网站。 相反，攻击者必须说服用户访问该网站，方法通常是让用户单击电子邮件或 Instant Messenger 消息中的链接以使用户链接到攻击者的网站。  
-   成功利用此漏洞的攻击者可以获得与本地用户相同的用户权限。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。  
-   在基于电子邮件利用此漏洞的攻击情形中，以纯文本格式阅读电子邮件的客户受此漏洞的威胁较小。
  
#### Microsoft DirectX 在 WAV 和 AVI 文件分析中的代码执行漏洞 (CVE-2007-3895) 的变通办法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。 Microsoft 已测试了以下变通方法，并在讨论中指明了变通方法是否会降低功能性：
  
-   **修改 quartz.dll 的访问控制列表**
  
    在 Windows XP（所有版本）上，通过命令提示符运行以下命令：
  
    `Echo y| Cacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /E /P everyone:N`
  
    在 Windows Vista（所有版本）上，通过提升的命令提示符运行以下命令：
  
    `Takeown.exe /f %WINDIR%\SYSTEM32\QUARTZ.DLLIcacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /save %TEMP%\QUARTZ_ACL.TXTIcacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /deny everyone:(F)`
  
**变通办法的影响：**  WAV 和 AVI 文件将无法在 Windows Vista 上的启用 DirectX 的应用程序中播放。 所有文件将无法在 Windows XP 上的启用 DirectX 的应用程序中播放 **。**
  
**如何撤消变通方法：** 在 Windows XP（所有版本）上，通过命令提示符运行以下命令：
  
`Cacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /E /R everyone`
  
在 Windows Vista（所有版本）上，通过提升的命令提示符运行以下命令：
  
`Icacls.exe %WINDIR%\SYSTEM32\QUARTZ.DLL /grant everyone:(F)Icacls.exe %WINDIR%\SYSTEM32 /restore %TEMP%\QUARTZ_ACL.TXT`
  
-   **注销 quartz.dll**
  
    `Regsvr32.exe –u %WINDIR%\SYSTEM32\QUARTZ.DLL`
  
**变通办法的影响：**  WAV 和 AVI 文件将无法在 Windows Vista 上的启用 DirectX 的应用程序中播放。 所有文件将无法在 Windows XP 上的启用 DirectX 的应用程序中播放。
  
**如何撤消变通方法：** 通过提升的命令提示符运行以下命令：
  
`Regsvr32.exe %WINDIR%\SYSTEM32\QUARTZ.DLL`
  
#### Microsoft DirectX 在 WAV 和 AVI 文件分析中的代码执行漏洞 (CVE-2007-3895) 的常见问题
  
**此漏洞的影响范围有多大？**    
这是一个远程执行代码漏洞。 成功利用此漏洞的攻击者可以完全远程控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**造成漏洞的原因是什么？**    
DirectShow 是集成的 DirectX 技术，不执行对 WAV 和 AVI 文件参数的充分验证。
  
**什么是 DirectShow？**    
Microsoft DirectShow 用于 Microsoft Windows 操作系统上的流式媒体。 DirectShow 用于高质量地捕获和播放多媒体流。 它自动检测和使用视频和音频加速硬件（如果有），同样也支持没有加速硬件的系统。 它还与其他 DirectX 技术集成。 可以使用 DirectShow 创建的某些类型的应用程序包括 DVD 播放器、视频编辑应用程序、AVI 到 ASF 转换器、MP3 播放器和数字视频捕获应用程序。
  
**什么是 DirectX？**    
Microsoft DirectX 是 Windows 操作系统的一个功能。 它用于 Microsoft Windows 操作系统上的流式媒体，在玩游戏或观看视频时支持图形和声音。
  
**攻击者可能利用此漏洞执行什么操作？**    
如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**攻击者如何利用此漏洞？**    
利用此漏洞需要用户打开特制格式文件。 但是，由于此漏洞在 Microsoft Windows 的流式组件中，攻击可以从特制网站或提供 Web 内容的应用程序发起。
  
**受此漏洞威胁最大的系统有哪些？**    
利用此漏洞需要用户打开特制格式文件。 但是，由于此漏洞可以由 Windows Media Player 或具有基于 Web 的播放功能的其他媒体播放器触发，攻击可以从特制网站或提供 Web 内容的应用程序发起。
  
**此更新有什么作用？**    
此更新通过增加 DirectShow 分析器对受支持文件类型执行的验证来消除此漏洞。

    1.  “**开始**”，然后单击“**搜索**”。
    2.  “**搜索结果**”窗格，单击“**搜索助理**”下面的“**所有文件和文件夹**”。

    4.  **属性**。

5.  在“**版本**”选项卡上，通过将计算机上安装的文件与相应文件信息表中记录的版本进行比较来确定该文件的版本。

    由于 Microsoft Windows 有多个版本，因此以下步骤可能会与您的计算机上的步骤不同。 如果不同，请参阅产品文档来完成这些步骤。

1.  单击“**开始**”，然后单击“**搜索**”。  
2.  在“**搜索结果**”窗格中，单击“**搜索助理**”下的“**所有文件和文件夹**”。

    4.  在文件列表中，右键单击相应文件信息表中的一个文件名，然后单击“**属性**”。
        
**注意** 由于安装的操作系统或程序的版本不同，因此文件信息表中所列的某些文件可能并未安装。
5.  在“**版本**”选项卡上，通过将计算机上安装的文件与相应文件信息表中记录的版本进行比较来确定该文件的版本。
        
**注意** 在安装过程中，除文件版本以外的其他属性均可能会发生变化。 不支持通过将其他文件属性与文件信息表中的信息进行比较这种方法来验证是否应用了更新。 另外，在某些情况下，在安装过程中还可能会重命名文件。 如果没有文件或版本信息，请使用另外一种方法来验证更新安装。
