Microsoft 安全公告 MS08-036 - 重要
==================================

实际通用多播 (PGM) 中的漏洞可能允许拒绝服务 (950762)
----------------------------------------------------

发布时间: 2008年6月10日

**版本:** 1.0

### 一般信息

#### 摘要

此安全更新可解决实际通用多播 (PGM) 协议中两个秘密报告的漏洞，如果受影响系统接收了格式错误的 PGM 数据包，则该漏洞可能允许拒绝服务。 成功利用此漏洞的攻击者可能导致用户系统停止响应，并且可能要求重新启动才能还原功能。 请注意，虽然攻击者无法利用拒绝服务漏洞来执行代码或提升他们的用户权限，但此漏洞可能导致受影响的系统停止接受请求。

对于 Windows XP 和 Windows Server 2003 的所有受支持版本，此安全更新等级为“重要”；对于 Windows Vista 和 Windows Server 2008 的所有受支持版本，此安全更新等级为“中等”。有关详细信息，请参阅本节中的“**受影响和不受影响的软件**”小节。

该安全更新通过修改 PGM 分析格式错误的数据包的方式来解决这些漏洞。 有关漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的常见问题 (FAQ) 小节。

**建议。**  Microsoft 建议客户尽早应用此更新。

**已知问题。** 无

PGM 无效的长度漏洞 - CVE-2008-1440  
----------------------------------
  
  
Microsoft Windows XP 和 Windows Server 2003 上的实际通用多播 (PGM) 协议实现中存在一个拒绝服务漏洞。该漏洞是由于不正确地验证特制 PGM 数据包引起的。 成功利用此漏洞的攻击者可能导致计算机无响应，并且要求重新启动才能还原功能。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2008-1440](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1440)。
  
#### PGM 无效的长度漏洞 (CVE-2008-1440) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   在运行 Windows XP 和 Windows Server 2003 的系统上，只有当安装了 Microsoft Message Queuing (MSMQ) 3.0 时，才会启用实际通用多播 (PGM)。 默认情况下不安装 MSMQ 服务。
  
#### PGM 无效的长度漏洞 (CVE-2008-1440) 的变通办法
  
Microsoft 并未发现此漏洞的任何变通办法。
  
#### PGM 无效的长度漏洞 (CVE-2008-1440) 的常见问题
  
**此漏洞的影响范围有多大？**    
这是一个拒绝服务漏洞。 利用此漏洞的远程匿名攻击者可能导致受影响的系统无响应，直至系统重新启动。 请注意，虽然攻击者无法利用拒绝服务漏洞来执行代码或提升他们的用户权限，但此漏洞可能导致受影响的系统停止接受请求。
  
**造成漏洞的原因是什么？**    
该协议的分析代码不正确地验证 PGM 数据包的选项长度字段。 如果受影响的系统收到具有无效的选项长度字段的特制 PGM 数据包，则分析程序会进入一个无限循环，并且最终导致系统无不响应。
  
**什么是实际通用多播 (PGM)？**  
PGM 是一个可靠且可伸缩的多播协议，通过该协议，接收方可以检测丢失、请求重新传输丢失的数据或者通知应用程序无法恢复的丢失情形。 PGM 是一个接收方可靠协议，这意味着接收方有责任确保收到所有数据，从而免除了发送方对于通信可靠性的责任。 PGM 适用于要求从多个源到多个接收方进行复制自由的多播数据传送的应用程序。 PGM 不支持确认传送，也不保证来自多个发送方的数据包的排序。 有关 PGM 的详细信息，请参阅下列 [MSDN 文章](http://msdn.microsoft.com/en-us/library/ms740125(vs.85).aspx)。
  
**什么是 Microsoft 消息队列？**  
Microsoft Message Queuing (MSMQ) 是 Windows 操作系统的一个组件，允许协作应用程序互相发送和接收消息，即使预期接收应用程序未在运行或者发送或接收应用程序所运行的计算机从网络断开连接也是如此。 消息会被存储下来并由 MSMQ 进行转发，直至到达目标队列。 随后，当接收应用程序运行时，它可以从队列中检索消息。 MSMQ 分离发送和接收应用程序，因此它们不需要同时运行。 消息队列提供内置的增强安全性、事务处理支持以及其他功能。 有关 MSMQ 的详细信息，请参阅 [MSMQ： 常见问题](http://www.microsoft.com/windowsserver2003/techinfo/overview/msmqfaq.mspx)。
  
**攻击者可能利用此漏洞执行什么操作？**    
成功利用此漏洞的攻击者可能导致用户系统停止响应，并且可能要求重新启动才能还原功能。
  
**攻击者如何利用此漏洞？**    
攻击者可能通过向受影响的系统发送特制 PGM 数据包来试图利用该漏洞。 然后，数据包可能导致受影响的系统无响应，直至重新启动。
  
**受此漏洞威胁最大的系统有哪些？**    
安装了 MSMQ 或者运行 PGM 支持的应用程序（侦听 PGM 数据包）的工作站和服务器受到的威胁最大。
  
**此更新有什么作用？**    
该更新通过正确地验证 PGM 数据包的选项长度字段来消除该漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**    
否。 Microsoft 通过可靠的披露渠道了解到有关此漏洞的信息。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已公开披露的信息。
  
**在发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**    
否。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已被公开用于攻击客户的信息，也没有看到任何发布的概念代码证明示例。
  
PGM 格式错误的片段漏洞 - CVE-2008-1441  
--------------------------------------
  
  
Microsoft Windows XP、Windows Server 2003、Windows Vista 和 Windows Server 2008 上的实际通用多播 (PGM) 协议实现中存在一个拒绝服务漏洞。该协议的分析代码无法正确地验证特制的 PGM 片段，将导致受影响的系统无响应，直至攻击停止。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2008-1441](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1441)。
  
#### PGM 格式错误的片段漏洞 (CVE-2008-1441) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   在运行 Windows XP 或 Windows Server 2003 的系统上，只有当安装了 Microsoft Message Queuing (MSMQ) 3.0 时，才会启用实际通用多播 (PGM)。 默认情况下不安装 MSMQ 服务。  
-   在运行 Windows Vista 或 Windows Server 2008 的系统上，只有当安装了 Microsoft Message Queuing (MSMQ) 4.0 时，才会启用实际通用多播 (PGM)。 默认情况下不安装 MSMQ 服务。 当安装了 MSMQ 服务时，PGM 协议可用，但默认情况下不启用。
  
#### PGM 格式错误的片段漏洞 (CVE-2008-1441) 的变通办法
  
Microsoft 并未发现此漏洞的任何变通办法。

该协议的分析代码无法正确地处理格式错误并且包含无效的片段选项的 PGM 数据包。

PGM 是一个可靠且可伸缩的多播协议，通过该协议，接收方可以检测丢失、请求重新传输丢失的数据或者通知应用程序无法恢复的丢失情形。 PGM 是一个接收方可靠协议，这意味着接收方有责任确保收到所有数据，从而免除了发送方对于通信可靠性的责任。 PGM 适用于要求从多个源到多个接收方进行复制自由的多播数据传送的应用程序。 PGM 不支持确认传送，也不保证来自多个发送方的数据包的排序。 有关 PGM 的详细信息，请参阅下列 [MSDN 文章](http://msdn.microsoft.com/en-us/library/ms740125(vs.85).aspx)。
  
**什么是 Microsoft 消息队列？**  
Microsoft Message Queuing (MSMQ) 是 Windows 操作系统的一个组件，允许协作应用程序互相发送和接收消息，即使预期接收应用程序未在运行或者发送或接收应用程序所运行的计算机从网络断开连接也是如此。 消息会被存储下来并由 MSMQ 进行转发，直至到达目标队列。 随后，当接收应用程序运行时，它可以从队列中检索消息。 MSMQ 分离发送和接收应用程序，因此它们不需要同时运行。 消息队列提供内置的增强安全性、事务处理支持以及其他功能。 有关 MSMQ 的详细信息，请参阅 [MSMQ： 常见问题](http://www.microsoft.com/windowsserver2003/techinfo/overview/msmqfaq.mspx)。
  
**攻击者可能利用此漏洞执行什么操作？**    
成功利用此漏洞的攻击者可能导致用户系统无响应。
  
**攻击者如何利用此漏洞？**    
攻击者可能通过向受影响的系统发送特制 PGM 数据包来试图利用该漏洞。 然后，数据包可能导致受影响的系统无响应。 当所有特制数据包处理完毕或者系统重新启动时，系统将返回到正常功能。 重新启动后，如果攻击者继续发送特制的 PGM 数据包，则系统将再次无响应。
  
**受此漏洞威胁最大的系统有哪些？**    
安装了 MSMQ 或者运行 PGM 支持的应用程序（侦听 PGM 数据包）的工作站和服务器受到的威胁最大。
  
**此更新有什么作用？**    
该更新通过正确地验证 PGM 数据包的片段选项来消除该漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**    
否。 Microsoft 通过可靠的披露渠道了解到有关此漏洞的信息。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已公开披露的信息。
  
**在发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**    
否。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已被公开用于攻击客户的信息，也没有看到任何发布的概念代码证明示例。
