Microsoft 安全公告 MS08-062 - 重要
==================================

Windows Internet 打印服务中的漏洞可能允许远程执行代码 (953155)
--------------------------------------------------------------

发布时间: 2008年10月14日

**版本:** 1.0

### 一般信息

#### 摘要

此更新解决了 Windows Internet 打印服务中一个秘密报告的漏洞，该漏洞可能允许在当前用户的上下文中远程执行代码。 如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。

对于 Microsoft Windows 2000、Windows XP、Windows Server 2003、Windows Vista 和 Windows Server 2008 的所有受支持版本，此更新的等级为“重要”。有关详细信息，请参阅本节的“**受影响和不受影响的软件**”小节。

此安全更新通过更改 Internet 打印协议 (IPP) 服务中分配内存的方式来解决此漏洞。 有关漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的常见问题 (FAQ) 小节。

**建议。** Microsoft 建议客户尽早应用此更新。

**已知问题。** 无

IPP 服务整数溢出漏洞 - CVE-2008-1446  
------------------------------------
  
  
运行 IIS 且启用了 Internet 打印服务的 Windows 系统上存在一个远程执行代码漏洞。 此问题可能允许经过身份验证的远程攻击者在受影响的系统上执行任意代码。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446)。
  
#### IPP 服务整数溢出漏洞 (CVE-2008-1446) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   对于 Microsoft Windows 2000 和 Windows XP，IIS 上默认情况下会启用 Internet 打印协议 (IPP)。 但是，所有平台上的默认文件权限已设置，因此只有“经过验证”组中的用户才能访问 IPP。 由于此限制，未经身份验证的客户默认情况下不具备远程利用此漏洞的必要文件夹权限。  
-   Windows Server 2003 上的 IIS 安装使用默认配置时不易受到攻击。 只有在 IIS 中启用 Internet 打印协议 ISAPI 筛选器的 Windows Server 2003 安装易受攻击。 “Internet 打印”和 Active Server Pages 组件对于启用基于 Web 的打印机管理和 Internet 打印是必需的。 如果 Active Server Pages 组件设置为“允许”，且 Internet 打印组件设置为“禁止”，则只启用了基于 Web 的打印机管理。 用户无法通过“Internet 打印”连接到任何共享打印机。  
-   默认情况下，Windows Server 2008 上的 IIS 安装不易受攻击。 运行 Windows Server 2008 的计算机可充当使用 Internet 打印的客户端计算机。 但是，提出打印请求的计算机用户，必须在经过打印服务器的身份验证之后才能使用连接到打印服务器的打印机。 要使运行 Windows Server 2008 的计算机能够充当支持 Internet 打印的服务器，必须安装 Web 服务器 (IIS) 角色、打印服务角色以及打印服务角色中的 Internet 打印角色服务。  
-   对此漏洞的利用要求 IIS 服务器向端口 445 上的恶意计算机进行出站通信。阻止从 IIS 服务器到 Internet 的出站 SMB 连接的管理员可防止从不信任系统对此漏洞的利用。
  
#### IPP 服务整数溢出漏洞 (CVE-2008-1446) 的变通方法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。 Microsoft 已测试了以下变通方法，并在讨论中指明了变通方法是否会降低功能性：
  
-   **禁用 IPP 服务**
  
    在 IIS 5.0 和 IIS 5.1 中禁用 IPP 服务
  
    1.  打开 **Internet 服务管理器**。  
    2.  右键单击“**默认网站**”，并选择“**属性**”。  
    3.  转到“**主目录**”选项卡并选择“**配置**”。  
    4.  向下滚动以查找 **.printer** 扩展名以及 **msw3prt.dll** 的相应路径。  
    5.  单击“**删除**”。  
    6.  单击“**确定**”。  
    7.  重新启动 IIS 服务。
  
    **注意** Internet 打印功能可以通过组策略以及 Internet 服务管理器进行配置。 如果组策略设置与 Internet 服务管理器中的设置冲突，组策略设置优先。 如果您通过 Internet 服务管理器删除 Internet 打印功能，请确保验证该功能未被本地或域组策略重新启用。 （默认组策略既不启用也不禁用 Internet 打印功能。） 在 MMC 组策略管理单元中，选中“计算机配置 | 管理模板 | 打印 | 基于 Web 的打印”。
  
    在 IIS 6.0 中禁用 IPP 服务
  
    1.  启动 **IIS 管理器**或 **IIS 管理单元**。  
    2.  展开 **\* server\_name**，其中 server\_name 是服务器的名称。  
    3.  单击“**Web 服务扩展**”。  
    4.  在右窗格中，单击“**Internet 打印**”，然后单击“**禁止**”。  
    5.  退出 IIS 管理器。
  
    在 Windows Server 2008 上的 IIS 7.0 中禁用 IPP 服务
  
    1.  如果**服务器管理器**未在运行，请依次单击“**开始**”、“**管理工具**”和“**服务器管理器**”。 （如果出现“**用户帐户控制**”对话框，请确认它显示的操作是您所需要的，然后单击“**继续**”。）  
    2.  展开“角色”节点，然后右键单击“**打印服务**”并选择“**删除角色服务**”。  
    3.  删除角色服务向导将出现。 如果“**Internet 打印**”选项被选中，请清除该复选框，单击“**下一步**”，然后单击“**删除**”以完成向导。
  
    **变通办法的影响。** 用户将无法通过“Internet 打印”连接到任何共享打印机。
  
    **如何撤消变通方法。**
  
    在 IIS 5.0 和 IIS 5.1 中启用 IPP 服务
  
    1.  打开 **Internet 服务管理器**。  
    2.  右键单击“**默认网站**”，并选择“**属性**”。  
    3.  转到“**主目录**”选项卡并选择“**配置**”。  
    4.  向下滚动以查找 **.printer** 扩展名以及 **msw3prt.dll** 的相应路径。  
    5.  单击“**Add**（添加）”。  
    6.  在标记为“**可执行文件**”的字段中，输入以下内容：  
        %systemroot%\\system32\\msw3prt.dll  
    7.  在标记为“**扩展名**”的字段中，输入以下内容：  
        .printer  
    8.  在标记为“**限制为**”的字段中，输入以下内容：  
        GET,POST  
    9.  完成后，重新启动 IIS 服务。
  
    在 IIS 6.0 中启用 IPP 服务
  
    1.  启动 **IIS 管理器**或 **IIS 管理单元**。  
    2.  展开 **\* server\_name**，其中 server\_name 是服务器的名称。  
    3.  单击“**Web 服务扩展**”。  
    4.  在右窗格中，单击“**Internet 打印**”，然后单击“**允许**”。  
    5.  退出 IIS 管理器。
  
    在 IIS 7.0 中启用 IPP 服务
  
    1.  如果**服务器管理器**未在运行，请依次单击“**开始**”、“**管理工具**”和“**服务器管理器**”。 （如果出现“**用户帐户控制**”对话框，请确认它显示的操作是您所需要的，然后单击“**继续**”。）  
    2.  展开“角色”节点，然后右键单击“**打印服务**”并选择“**添加角色服务**”。  
    3.  添加角色服务向导将出现。 选中“**Internet 打印**”选项，单击“下一步”，然后单击“**安装**”以完成向导。
  
-   **运行 IIS 锁定工具 2.1**
  
    在运行 IIS 5.0 的 Microsoft Windows 2000 或运行 IIS 5.1 的 Windows XP 上，IIS 锁定向导版本 2.1 可用于禁用不必要的功能，包括 Internet 打印。 要运行此向导，请完成以下操作：
  
    1.  从[这里](https://www.microsoft.com/download/details.aspx?familyid=dde9efc0-bb30-47eb-9a61-fd755d23cdec&displaylang=en)下载 IIS 锁定工具 2.1。  
    2.  执行以下任一操作：  
        要立即开始安装，请单击从当前位置**打开**或**运行此程序**。  
        要将下载文件复制到您的计算机以便稍后安装，请单击“**保存**”或“**将该程序保存到磁盘**”。
  
    **变通办法的影响。**
  
    用户将无法通过“Internet 打印”连接到任何共享打印机。 另外，IIS 锁定工具用户将无法通过“Internet 打印”连接到任何共享打印机。 有关 IIS 锁定工具所做更改的完整列表，请参阅以下 [MSDN 文章](http://msdn.microsoft.com/en-us/library/aa302365.aspx)。
  
    **如何撤消变通方法。**
  
    要撤消 IIS 锁定所做的更改，请再次运行 **IISlockd.exe**，并选择撤消更改。 撤消操作可还原在您之前运行 IIS 锁定前立即生效的系统设置。 这些详细信息包含在日志文件 \\WINNT\\System32\\inetsrv\\0blt-log 中。 因此，在您运行 IIS 锁定之后立即测试系统，这一点非常重要。 如果需要撤消，请立即执行。
  
    **注意** 作为 IIS 锁定的一部分进行安装的 URLScan ISAPI 筛选器不会作为撤消过程的一部分被删除。 您可通过在“Internet 服务管理器”中使用服务器级别的 ISAPI 筛选器选项卡手动删除 URLScan。
  
#### IPP 服务整数溢出漏洞 (CVE-2008-1446) 的常见问题
  
**此漏洞的影响范围有多大？**      
这是一个远程执行代码漏洞。 如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**造成漏洞的原因是什么？**      
在运行 IIS 的 Windows 服务器上的 Microsoft Internet 打印协议 (IPP) 实施中存在一个远程的、经过身份验证的漏洞。 IPP 功能作为 IIS 的 ISAPI 扩展实施。 存在于 IPP 实施中的整数溢出可能允许经过身份验证的攻击者在受影响的 IIS 服务器上远程执行代码。
  
**什么是 ISAPI？**    
ISAPI（Internet 服务应用程序编程接口）是一项技术，使 Web 开发人员能够通过编写为 Web 服务器提供新服务的自定义代码来扩展其 Web 服务器功能。 如果新功能提供低级别服务，则自定义代码可在 ISAPI 筛选器中执行；如果新功能提供高级别服务，则自定义代码可在 ISAPI 扩展中执行。 在这种情况下，受影响的代码是 ISAPI 扩展。
  
**此处所说的 ISAPI 扩展是什么？**    
受影响的 ISAPI 扩展是一个实施 Internet 打印协议 (IPP) 的扩展，它是一个在 RFCs 2910 和 2911 中定义的行业标准。IPP 提供一种通过 HTTP 在 Internet 上请求打印服务并了解打印作业状态的方法。 例如，使用 IPP，漫游用户可通过 Internet 发送打印作业，并在公司网络的打印机上进行打印。 用户还可以了解打印请求是否已准确无误地完成。
  
Microsoft Windows 2000 引入了对 Internet 打印的内在支持。 Microsoft Windows 2000 实施使用户可以直接打印到 URL，并通过浏览器查看有关打印作业的信息。 在 Microsoft Windows 2000 Server 版本中，对 Internet 打印的支持默认为启用。 它还可作为一个可选组件安装在 Microsoft Windows 2000 Professional、Windows XP、Windows Server 2003 和 Windows Server 2008 上。
  
**攻击者可能利用此漏洞执行什么操作？**      
成功利用此漏洞的攻击者能够在用户系统上运行恶意代码。 如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
**攻击者如何利用此漏洞？**      
攻击者可发送特制的 HTTP POST 请求到易受攻击的 web 服务器，这可能导致该服务器连接到由攻击者控制的使用 IPP 充当打印机的计算机。 然后，攻击者使用特制的响应在目标服务器上远程执行代码。
  
**受此漏洞威胁最大的系统有哪些？**      
运行启用了 Internet 打印协议 (IPP) 服务的 IIS 的工作站和服务器受此漏洞的威胁最大。
  
**此更新有什么作用？**      
此安全更新通过验证从 Internet 打印协议 (IPP) 服务中的远程计算机接收的数据来解决该漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**      
否。 Microsoft 通过可靠的披露渠道了解到有关此漏洞的信息。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已公开披露的信息。
  
**发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**      
是，Microsoft 获悉尝试使用此漏洞的目标攻击。 但是，在最初发布此安全公告时，Microsoft 并未看到任何发布的概念证明代码示例。
