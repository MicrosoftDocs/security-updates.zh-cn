Security Bulletin

Microsoft 安全公告 MS11-006 - 严重
==================================

Windows Shell 图形处理中的漏洞可能允许远程执行代码 (2483185)
------------------------------------------------------------

发布时间: 2011年2月8日 | 更新时间: 2011年2月14日

**版本:** 1.1

### 一般信息

#### 摘要

此安全更新解决了 Windows Shell 图形处理器中一个公开披露的漏洞。 如果用户查看特制缩略图，此漏洞可能允许远程执行代码。 成功利用此漏洞的攻击者可以获得与登录用户相同的用户权限。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。

对于 Windows XP、Windows Server 2003、Windows Vista 和 Windows Server 2008 的所有受支持版本，此安全更新的等级为“严重”。Windows 7 和 Windows Server 2008 R2 的所有受支持版本不受此漏洞的影响。 有关详细信息，请参阅本节中“**受影响和不受影响的软件**”小节。

此安全更新通过更正 Windows Shell 图形处理器解析缩略图的方式解决此漏洞。 有关漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的常见问题 (FAQ) 小节。

此安全更新也解决了最初在 [Microsoft 安全通报 2490606](http://technet.microsoft.com/security/advisory/2490606) 中描述的漏洞。

**建议。** 大多数客户均启用了“自动更新”，他们不必采取任何操作，因为此安全更新将自动下载并安装。 尚未启用“自动更新”的客户必须检查更新，并手动安装此更新。 有关自动更新中特定配置选项的信息，请参阅 [Microsoft 知识库文章 294871](http://support.microsoft.com/kb/294871)。

对于管理员、企业安装或者想要手动安装此安全更新的最终用户，Microsoft 建议客户使用更新管理软件立即应用此更新或者利用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服务检查更新。

另请参阅本公告后面部分中的“**检测和部署工具及指导**”一节。

**已知问题。** 无

Windows Shell 图形处理溢出漏洞 - CVE-2010-3970  
----------------------------------------------
  
<span></span>  
Windows Shell 图形处理器处理特制缩略图的方式中存在远程代码执行漏洞。 成功利用此漏洞的攻击者可以在登录用户的安全上下文中运行任意代码。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970)。
  
#### Windows Shell 图形处理溢出漏洞 (CVE-2010-3970) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   此漏洞无法通过电子邮件自动加以利用。  
-   成功利用此漏洞的攻击者可以获得与登录用户相同的用户权限。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
#### Windows Shell 图形处理溢出漏洞 (CVE-2010-3970) 的变通办法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。 Microsoft 已测试了以下变通方法，并在讨论中指明了变通方法是否会降低功能性：
  
-   **修改 Windows XP 和 Windows Server 2003 系统上 shimgvw.dll 的访问控制列表 (ACL)**
  
    **注意** 要使用自动化“**Microsoft 修复**”解决方案来启用或禁用此变通办法，请参阅 [Microsoft 知识库文章 2483185](http://support.microsoft.com/kb/2483185)。
  
    要修改 shimgvw.dll 的 ACL 以增加限制，请以管理员身份通过命令提示符运行以下命令：
  
    **对于 32 位版本的 Windows XP 和 Windows Server 2003：**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:N`
  
    **对于 64 位版本的 Windows XP 和 Windows Server 2003：**
  
    ```
        Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:N    
        Echo y| cacls %WINDIR%\SYSWOW64\shimgvw.dll /E /P everyone:N    
    ```    
  
    **变通办法的影响。** 通常由 Windows Shell 图形处理器处理的媒体文件将不会正确显示。 注意：在安装 MS11-006 安全更新之前，您必须从您之前应用此更新的系统中撤消此变通办法。
  
    **如何撤消变通方法。**
  
    以管理员身份通过命令提示符运行下列命令：
  
    对于 32 位版本的 Windows XP 和 Windows Server 2003：
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyone`
  
    对于 64 位版本的 Windows XP 和 Windows Server 2003：
  
    ```
        cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyone
        cacls %WINDIR%\SYSWOW64\shimgvw.dll /E /R everyone
    ``` 
-   **禁用查看 Windows Vista 和 Windows Server 2008 系统上 Windows 资源管理器中的缩略图**
  
    要禁用查看 Windows Vista 和 Windows Server 2008 上 Windows 资源管理器中的缩略图，请执行以下步骤：
  
    1.  打开“**控制面板**”，选择“**外观和个性化**”。  
    2.  从菜单中选择“**文件夹选项**”。  
    3.  选择“**查看**”选项卡，并确保“**始终显示图标，从不显示缩略图**”选项已被选中。 单击“**确定**”保存设置。  
    4.  关闭 Windows 资源管理器所有打开的实例以使更改生效。
  
    **变通办法的影响。** Windows 资源管理器将不会显示缩略图。
  
    **如何撤消变通方法。**
  
    要启用 Windows Vista 和 Windows Server 2008 中的缩略图，请执行以下步骤：
  
    1.  打开“**控制面板**”，选择“**外观和个性化**”。  
    2.  从菜单中选择“**文件夹选项**”。  
    3.  选择“**查看**”选项卡，并确保“**始终显示图标，从不显示缩略图**”选项未被选中。 单击“**确定**”保存设置。  
    4.  关闭 Windows 资源管理器所有打开的实例以使更改生效。
  
#### Windows Shell 图形处理溢出漏洞 (CVE-2010-3970) 的常见问题
  
**此漏洞的影响范围有多大？**    
这是一个远程执行代码漏洞。 成功利用此漏洞的攻击者可以完全远程控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**造成此漏洞的原因是什么？**    
此漏洞由 Windows Shell 图形处理器未正确解析特制缩略图导致，进而造成堆栈溢出。
  
**什么是 Windows Shell？**    
Windows 用户界面 (UI) 使用户有权访问运行应用程序和管理操作系统所必需的各种对象。 这些对象中，最多且最常见的是驻留在计算机硬盘中的文件夹和文件。 此外，还有大量虚拟对象，这些对象允许用户执行诸如向远程打印机发送文件或访问回收站等任务。 Shell 将这些对象组织成层次名称空间，并向用户和应用程序提供统一而有效的方式来访问和管理对象。
  
**攻击者可能利用此漏洞执行什么操作？**    
如果用户使用管理用户权限登录，成功利用此漏洞的攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
**攻击者如何利用此漏洞？**    
此漏洞要求用户查看特制缩略图。
  
在网络情形中，攻击者可能将特制缩略图或包含特制缩略图的文件放在网络共享上，例如 UNC 或 WebDAV 位置，然后诱使用户浏览到 Windows 资源管理器中的位置。 当用户导航到该共享时，受影响的控制路径通过 Windows Shell 图形处理器触发。 特制缩略图然后可以利用此漏洞并在已登录用户的安全上下文中执行代码。 攻击者无法强迫用户访问网络共享、UNC 或 WebDAV 位置。 相反，攻击者必须说服用户访问共享，方法通常是让用户单击电子邮件或 Instant Messenger 消息中的链接以使用户链接到网络共享、UNC 或 WebDAV 位置。
  
**什么是统一命名约定 (UNC)？**  
统一命名约定 (UNC) 是一种标识资源（如网络上的共享名称或文件）的方法。 典型的 UNC 名称是以两个反斜线开头，紧接着是服务器名称，例如 \\\\服务器\\共享\\子目录\\文件名。
  
**受此漏洞威胁最大的系统有哪些？**    
工作站和终端服务器受到的威胁最大。 如果管理员允许用户登录服务器并运行程序，服务器可能面临更大风险。 不过，最佳做法是不要授予这样的权限。
  
**此更新有什么作用？**    
此更新通过更正 Windows Shell 图形处理器解析缩略图的方式解决此漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**    
是。 此漏洞已公开披露。 已为此漏洞分配了常见漏洞和披露号码 [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970)。此漏洞已在 [Microsoft 安全通报 2490606](http://technet.microsoft.com/security/advisory/2490606) 中描述。
  
**在发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**    
否。 在最初发布此安全公告时，Microsoft 看到了公开发布的概念证明代码示例，但未收到任何表明此漏洞已被公开用于攻击用户的直接报告。
  
**应用此安全更新有助于客户防止尝试利用此漏洞的已公开发布代码吗？**    
是。 此安全更新将消除可能使用已发布的概念证明代码利用的漏洞。 已为消除的漏洞分配了常见漏洞和披露号码 [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970)。
